Example Skills Reference

Seven example SKILL.md files for the AI Product Development Lifecycle. Works with Cursor, Claude Code, and Gemini. Use these as-is or as examples to create your own versions of skills. See below for how to add them to your system.

Installation

Two ways to install: manually copy the files, or ask your LLM to do it for you.

C

Cursor

Manual Install

  • Create .cursor/skills/{skill-name}/SKILL.md for each skill
  • Copy the full SKILL.md content into each file
  • Restart Cursor to discover new skills
  • Invoke with /skill-name in chat

Cross-Tool Directory

  • Alternatively use .agents/skills/ — Cursor auto-discovers this path
  • This directory is also scanned by Claude Code and Gemini

Ask the LLM

Create a new Cursor skill at .cursor/skills/plan-tests/SKILL.md with the following content:

[paste SKILL.md content here]

Claude Code

Manual Install

  • Create .claude/skills/{skill-name}/SKILL.md for each skill
  • Copy the full SKILL.md content into each file
  • Skills are discovered automatically — no restart needed
  • Invoke with /skill-name in the terminal

Cross-Tool Directory

  • .agents/skills/ is also supported via symlink

Ask the LLM

Create a new skill at .claude/skills/plan-tests/SKILL.md with the following content:

[paste SKILL.md content here]
G

Gemini

Manual Install

  • Create .gemini/skills/{skill-name}/SKILL.md for each skill
  • Or use the cross-tool .agents/skills/ alias
  • Gemini auto-discovers skills based on description
  • Confirm activation when prompted
  • Manage with /skills list and /skills enable

Global Install

  • Place in ~/.gemini/skills/ for access across all projects

Design Principles

Every skill in this collection follows these four principles.

Infer, don't hardcode

Conventions are derived from the codebase at runtime, not maintained in static config. This keeps skills portable across projects.

Separate planning from implementation

Reviewing a plan is fast and high-signal. Reviewing generated code without a plan is slow and error-prone.

Automate context gathering

Business context comes from Jira MCP. Technical context comes from scanning the codebase. Engineers review output, not assemble input.

Think beyond functional tests

Always assess whether performance, load, or concurrency tests are warranted for the code under test.
